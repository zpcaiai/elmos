# Skill B37-X16: b37-marketplace-closure-certification-gate

## Use this skill when

- The Batch 37 extension ecosystem requires this lifecycle capability to be production-complete and independently testable.
- A Codex implementation task must create typed contracts, deterministic workflows, evidence, and conservative gates rather than prose-only policy.

## Domain-specific risks and invariants

- Core certification is necessary but not sufficient for ecosystem closure.
- No average score can offset a critical security, tenant, legal, financial, continuity, or data-portability failure.
- The gate cannot modify source evidence or self-approve a waiver.

## Workflow

1. Run the original Batch 37 core marketplace gate.
2. Run every closure schema and semantic validator against the exact Marketplace Pack.
3. Resolve all closure evidence references and verify holdout and representative lifecycle workflows.
4. Evaluate closure metrics, zero-tolerance findings, stale state, overdue certification, drift, incidents, legal/support blocks, settlement difference, and residual EOL dependencies.
5. Emit only certified, limited, experimental, or blocked status supported by exact evidence.

## Required repository outputs

- `certification/closure-gate-result.json`
- closure gate report, closure evidence graph, and final customer support profile

## Verification

- Run `run_marketplace_closure_gate.py`.
- Replay negative closure tests and verify fabricated status, missing evidence, stale mirror, unresolved incident, and non-zero settlement are rejected.

## Stop and escalate when

- Any required closure contract or independent evidence is missing.
- A P0 zero-tolerance condition remains non-zero.

## Definition of done

- Core and closure gates pass, all lifecycle states reconcile, and the exact pack receives the strongest evidence-supported final status.
