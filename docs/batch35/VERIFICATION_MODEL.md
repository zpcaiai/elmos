# Verification Model

The model is Claim → Risk → Required Techniques → Oracle Composition → Executions → Counterexamples → Coverage → Assurance. Techniques are complementary: example tests check known cases; property and metamorphic testing explore domains; mutation measures test strength; fuzzing finds robustness faults; symbolic and SMT methods reason about bounded paths; models verify workflows; schedule exploration targets concurrency; assurance records limitations and residual risk.
