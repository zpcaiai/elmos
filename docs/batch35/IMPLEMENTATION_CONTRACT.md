# Batch 35 Implementation Contract

Batch 35 implementations must use exact versioned verification packs. Every critical claim maps to one validation profile, at least two suitably independent oracles where required, executable specifications, immutable evidence, replayable counterexamples, and a conservative certification result. Formal, statistical, source-based, business, and model evidence must remain distinguishable. Unknown is never passed.

## Mandatory flow

1. Lock source/target artifacts, environment, route, workload, policies, and owners.
2. Define claims and risk tiers.
3. Register oracles and conflicts.
4. Implement executable techniques required by the profile.
5. Run negative, holdout, and representative workloads.
6. Minimize and replay every critical failure.
7. Build assurance case and residual-risk register.
8. Run the conservative gate.
