
# Cloud and DevOps capability matrix

Every pack should explicitly classify at least these domains:

- Container build and image supply chain
- Kubernetes or alternate orchestrator
- Helm or package templates
- Terraform or source IaC tool
- Cloud-native IaC templates
- CI/CD workflows and protected environments
- Configuration, secret, and environment promotion
- Human and workload identity
- Network, DNS, ingress, egress, and service mesh
- Serverless and event-driven runtime
- Managed database, cache, queue, stream, and object storage
- API gateway, load balancer, traffic policy, canary, and rollback
- Metrics, logs, traces, alerts, dashboards, and retention
- Security guardrails and policy as code
- Capacity, resilience, cost, tagging, and quota
- Drift, import, state ownership, destroy, and orphan detection

Statuses are `certified`, `supported`, `conditional`, `experimental`, `detected-only`, and `blocked`.
