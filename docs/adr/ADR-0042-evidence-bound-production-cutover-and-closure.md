# ADR-0042: Evidence-bound production cutover and closure

## Status

Accepted

## Context

Batch 10 can establish that an immutable target artifact is eligible for progressive delivery, but it
does not establish that production data moved, that every caller follows one write authority, that
traffic is safely routed, or that a legacy system may be retired. These activities cross operational,
business, security, compliance and irreversible data boundaries.

A single percentage flag is insufficient for stateful migration. Reads and writes can move at different
times; sessions and cohorts must be stable; backfill and CDC positions must be durable; messages, jobs,
files, caches and indexes have distinct closure semantics. A reported deployment or green dashboard is
not evidence that target truth, legacy drain and decommission all occurred.

## Decision

Add `modules/production-cutover` as an evidence-bound PCCM control plane for cross-language Batch 11.
It:

- admits only immutable Batch 10 P-E/P-F targets eligible for progressive delivery;
- records P0 through P12 and permits only adjacent phase transitions;
- separates read routing, write routing, source of truth and fallback authority by stable wave/cohort;
- enforces one primary writer, atomic authority publication, all-entry-point compliance and no client override;
- requires Expand–Migrate–Contract, resumable idempotent backfill, durable checkpoints and complete CDC;
- verifies every authoritative asset, final positions, reconciliation, messages, jobs, files, caches and indexes;
- maintains explicit rollback points, reverse synchronization, source capacity and an irreversibility frontier;
- requires continuous Hypercare and independent Business, Engineering, Security, Operations and Compliance acceptance;
- permits retirement only after traffic, writes, callers, consumers, jobs, archives, legal holds and approvals close;
- delegates all production observation and change to seven injected external authorities;
- produces an atomic, append-only, symbolic-link-safe evidence tree outside source and target repositories;
- reserves `migrationCompleted=true` exclusively for C-G with target truth and externally evidenced decommission.

The state machine returns authorization only. `productionChangeExecuted=false` is invariant in the
control-plane decision. The module cannot invoke host processes, route production traffic, mutate
customer data, revoke credentials or destroy infrastructure.

## Consequences

Local tests can prove deterministic policy behavior, fail-closed authority handling, phase safety and
artifact construction without production access. They cannot prove a migration, cutover or retirement
occurred. Real evidence must be generated by named, separately authorized systems and bound to the exact
run and immutable target artifact.

The stricter model increases preparation and evidence cost. In exchange, it makes rollback capability,
data closure, acceptance and decommission auditable, prevents phase skipping and dual-primary drift, and
keeps “target deployed,” “target is system of record,” “legacy drained,” and “migration completed” as
distinct claims. Missing production evidence remains `NOT_RUN`, blocked or inconclusive.
