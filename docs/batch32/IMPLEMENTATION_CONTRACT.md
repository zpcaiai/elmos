# Batch 32 implementation contract

## Purpose

Batch 32 turns frontend, server-rendered UI, desktop, and mobile modernization into exact, directional, evidence-backed client packs. A pack does not certify an entire framework family. It certifies a declared source tuple, target tuple, provider combination, browser or device matrix, rendering strategy, route and component scope, P0 user journeys, and quality profile.

## Mandatory implementation order

1. Capture an immutable source snapshot and exact source and target tuples.
2. Discover static and runtime routes, views, components, state, effects, forms, API clients, identity, assets, design systems, accessibility, rendering, tests, browsers, devices, and native boundaries.
3. Emit typed UI Interaction IR with stable IDs, source traces, explicit unknowns, and obligations.
4. Select a versioned target profile. Target generators consume UI IR and target profile, not source framework syntax.
5. Implement the smallest production-shaped vertical user journey.
6. Build and launch real source and target applications.
7. Execute route, interaction, state, form, API, identity, visual, accessibility, localization, rendering, performance, and business observations.
8. Run negative, holdout, and representative journeys.
9. Generate digested evidence and run the conservative gate.

## Non-negotiable constraints

- Never certify from screenshots, generated HTML, component stories, static analysis, or one browser alone.
- Never use status edits, baseline updates, screenshot masks, tolerance changes, `any`, suppressions, or disabled checks as substitutes for evidence.
- Source and target state, permissions, locale, theme, time, network, feature flags, and external responses must be controlled for comparison.
- Visual similarity cannot override accessibility, permission, form, data, error, privacy, or business correctness.
- Customer-owned target code, protected regions, design assets, and UX decisions must not be overwritten.
- Production captures must be privacy-minimized and must not contain credentials or unapproved personal or customer content.

## Certification scope

Certification is bounded by exact source and target versions, build and runtime tuple, provider profile, route and component scope, rendering strategy, browser and device matrix, locale and theme matrix, P0 journeys, and evidence. Reverse routes, alternative targets, other versions, devices, or untested journeys are not implied.
