
# ELMOS Product Convergence & Reference Implementation

本目录将Batch 1–45能力收敛为统一产品内核、Reference Route和真实客户实施计划。它不是新增功能Batch，而是实施与治理覆盖层。

## 核心原则

1. 统一Capability Package、Dependency Graph、Workflow、Evidence、Policy与Skill Registry。
2. 产品内核稳定，语言/框架/数据库/云/行业能力通过Pack扩展。
3. 一条Java/Spring → C#/ASP.NET Reference Route优先于继续扩路线。
4. Control Plane模块化单体与Private Runner优先于高级Agent自治。
5. Build Green不等于完成；必须同时通过Behavior、Security、Production、Maintainability和Customer Handoff。
6. 核心指标为Verified Migrated Workload，而不是代码行数。
