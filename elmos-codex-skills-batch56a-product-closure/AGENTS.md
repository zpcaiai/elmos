# ELMOS Product Closure Rules

1. Freeze uncontrolled horizontal Batch expansion.
2. Use the canonical Kernel and System of Record Matrix.
3. Do not equate Skill presence with implementation.
4. Do not equate static validation with runtime certification.
5. Activate only a bounded, explainable Skill set per task.
6. Every side effect needs a plan, typed permission, idempotency and evidence.
7. UNKNOWN_RESULT requires reconciliation before retry.
8. P0 failures cannot be waived.
9. Planning-edition Skills cannot be represented as GA.
10. Product progress is measured by Golden Journeys, providers, reliability, economics and customer acceptance.
