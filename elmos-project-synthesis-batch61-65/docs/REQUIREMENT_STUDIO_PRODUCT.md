# Requirement Studio Product Surfaces

1. Requirements — conversation, sources, glossary, requirements, assumptions, questions, conflicts, scope, priority, risk, acceptance, and baseline.
2. Architecture — context, containers, modules, data, communication, threats, observability, deployment, ADRs, and approvals.
3. Project Preview — repository tree, ownership, traceability, semantic diff, contract/data/deployment change, and merge conflicts.
4. Build & Run — stages, logs, traces, tests, environment matrix, diagnosis, repair limits, pause/cancel/retry, and evidence.
5. Evidence — requirement-to-release lineage, signatures, hashes, exports, and certification.
6. Administration — tenant policies, models, tools, registries, runners, quotas, retention, usage, and billing allocation.
7. Marketplace — Skills, templates, profiles, and Domain Packs with provenance, certification, support, and deprecation.
8. Support & Learning — privacy-safe diagnostic bundles, feedback triage, regression cases, and governed Skill improvements.
