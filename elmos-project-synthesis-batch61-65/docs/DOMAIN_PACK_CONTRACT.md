# Domain Pack Contract

Every Domain Pack contains:

- Identity, version, publisher, compatibility, signature, and license
- Domain glossary and aliases
- Entities, value objects, aggregates, commands, queries, and events
- Business rules, workflows, permissions, data classifications, and integration contracts
- Architecture guidance and quality-attribute defaults
- Positive, negative, boundary, security, failure, and migration tests
- Language mapping guidance for Java, Python, and .NET
- Explicit extension points and override precedence
- Compliance and safety boundaries

Domain Packs cannot weaken platform or tenant policy and cannot directly modify core engine behavior.
