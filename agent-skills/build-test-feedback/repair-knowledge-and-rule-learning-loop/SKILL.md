# Repair Knowledge and Rule Learning Loop
Read `../references/batch-8-repair-loop.md`. Store diagnostic/root-cause patterns, recipes, dependency/mapping/test/fixture fixes, failed attempts and anti-patterns with language/framework/version scope and validation statistics.

Promote a candidate only after clear causality, minimality, idempotence, multiple fixtures/repositories, regression success and human review. Keep private code out of public rules, preserve negative history and rerun the regression corpus for rule updates.
