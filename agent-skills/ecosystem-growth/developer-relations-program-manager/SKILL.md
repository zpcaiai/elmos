* DevRel不只是营销活动；
* 工程和产品需持续参与；
* 社区反馈必须有产品入口；
* DevRel指标不能只看Follower；
* 技术主张必须真实；
* 开源承诺需长期维护；
* 地区DevRel需本地语言和文化能力。

* 产品反馈进入Roadmap；
* API和SDK采用提高；
* 社区贡献增加；
* DevRel推动高质量使用；
* 技术品牌可信度提高。
