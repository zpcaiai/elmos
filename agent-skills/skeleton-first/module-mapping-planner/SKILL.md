# Module Mapping Planner
Read `../references/skeleton-v1.md`. Combine build and semantic dependency graphs, APIs, SCCs, tests, responsibilities and migration modes. Record one-to-one, merge, split, retain, externalize, or drop-with-proof with rules/rationale/risk. Recompute target dependencies/cycles and parallel waves. Do not map by directory alone, create an unbounded shared bucket, lose tests, or drop without proof.
