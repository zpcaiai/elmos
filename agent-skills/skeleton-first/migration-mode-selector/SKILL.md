# Migration Mode Selector
Read `../references/skeleton-v1.md`. Use UIR/opaque/dynamic/test/criticality/performance/team/dependency/license/timeline evidence. Each module gets one primary recorded mode and risks. High-dynamic low-test code cannot auto-select idiomatic rewrite; native/ML may need wrapper/retention; finance/transaction/auth favor fidelity; compatibility dependencies require audit. Retained/wrapper modes require explicit interop boundaries.
