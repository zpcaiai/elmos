# Target Dependency Candidate Discovery
Read `../references/dependency-migration-v1.md`. Search only configured target runtime catalogs, approved registries, reviewed knowledge, organization catalogs and explicit boundary capabilities. Attach exact version, platform, support, provenance, API coverage hints and strategy to each candidate. Preserve rejected candidates and reasons. Package-name similarity, download counts, descriptions, embeddings or LLM opinion are discovery signals only; they cannot create an approved candidate or bypass evidence and target policy.
