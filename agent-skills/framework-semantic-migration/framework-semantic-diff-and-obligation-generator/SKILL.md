# Framework Semantic Diff and Obligation Generator
Read `../references/afsm-v1.md`. Compare route, binding, serialization, pipeline order, DI scope, lifecycle, transactions, ORM, security, configuration, messaging, cache, scheduling and health at entity level.

Accept a difference only with explicit verified-equivalence evidence. Otherwise create a stable blocking obligation with risk, owner/waiver fields and verification strategy. Keep waivers auditable and require owner plus reason; never hide differences only in logs.
