# Unified Security Delivery Integration

1. Reuse shared Organization, Repository, Landscape, Portfolio, Plan, Step, Risk, Approval, Evidence, Delivery Snapshot, Audit, and Billing authorities.
2. Bind security extensions to schema, artifact reference, commit, artifact digest, deployment, infrastructure state, identity configuration, data classification, threat model, catalog, policy, and tool versions.
3. Publish separate Identity, Secrets, Cryptography, Threat Model, SAST, SCA, DAST, API, SBOM, Provenance, Vulnerability, Cloud, Data Protection, Control Assessment, and Authorization checks.
4. Compute the composite gate from repository, artifact, runtime, infrastructure, data, control, and authorization evidence; one passing scanner never establishes security.
5. Audit privilege, rotation, signing, VEX, suppression, false positive, risk acceptance, tailoring, assessment, authorization, suspension, and break-glass events.
6. Package estate, identity, secrets, crypto, threats, supply chain, scans, vulnerabilities, cloud, data, detections, controls, OSCAL, and authorization for offline verification.

## Acceptance

Deliver only evidence-backed internal decisions. Preserve exact `NOT_RUN`, `PARTIALLY_RUN`, `INCONCLUSIVE`, `TOOL_ERROR`, and `COVERAGE_INSUFFICIENT` states and never imply external certification.
