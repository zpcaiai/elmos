# Suite Target Planner

## Plan

1. Pin the Suite Estate, business capability map, process variants, customization inventory, master data, history, integrations, security, regulations, licensing, roadmap, and change capacity.
2. Generate at least two feasible target candidates; do not default every customer to cloud SaaS.
3. Separate technical upgrade or conversion from process redesign and greenfield implementation.
4. Evaluate selective transition, long-lived coexistence, module replacement, and best-of-breed composition.
5. Compare business fit, history scope, country constraints, availability, scale, team, cost, vendor support, and Cutover window.
6. Build waves by company, country, plant, business unit, module, process, user cohort, or data domain.

## Gate

Require a named target authority, operating-model approval, master-data and integration ownership, organizational change readiness, rollback feasibility, and explicit blockers when no option is feasible.

## Output

Produce ranked candidates, assumptions, rejected alternatives, wave DAG, target profile, risks, gates, and evidence references.
