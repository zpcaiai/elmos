# Mainframe Unified ELMOS Integration

## Integrate the change set

1. Emit `scope=MAINFRAME`, `engine=ELMOS_MAINFRAME`, a versioned extension schema, immutable artifact reference, tenant, source snapshot, runtime window, and engine version.
2. Link mainframe assets to Java, .NET, Python, database, frontend, infrastructure, security, test-quality, and composite plans through copybook, API, data, transaction, batch, identity, and journey dependencies.
3. Map copybook ambiguity, dynamic calls, missing restart, COMMAREA loss, IMS position, semantic differences, and unknown consumers into shared risks.
4. Keep online transaction, batch, data-authority, semantic-equivalence, parallel-run, cutover, and decommission checks independent.
5. Record discovery, analysis, z/OS CPU/test, job execution, transformation, validation, parallel-run, and cutover cost units without treating billing records as quality evidence.

## Package evidence

- Include source and load-module lineage, copybook/JCL versions, rule authority, approvals, job/spool hashes, test and comparison results, route/writer changes, scheduler changes, and access revocation.
- Make the evidence pack content-addressed and offline-verifiable.
- Never let an engine worker modify its own gate, approval, route, or authoritative business rule.
