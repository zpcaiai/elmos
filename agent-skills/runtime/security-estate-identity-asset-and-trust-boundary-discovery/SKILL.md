# Security Estate Discovery

1. Ingest the approved System Landscape and inventory code, artifacts, identities, clients, data, infrastructure, build systems, admin paths, and third parties.
2. Record asset owner, criticality, region, exposure, consumers, monitoring, and immutable evidence.
3. Model data flows with source, destination, classification, purpose, protocol, encryption, retention, and crossed trust boundaries.
4. Mark controls `DECLARED`, `OBSERVED`, `VERIFIED`, `PARTIAL`, or `UNKNOWN`; configuration presence alone is never verification.
5. Classify public and private exposure explicitly and promote unknown identity, endpoint, secret store, flow, or owner states into formal risk.

## Output

Emit versioned estate, assets, identities, boundaries, attack surface, data flows, existing controls, and unknowns bound to source and environment evidence.
