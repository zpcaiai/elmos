# Compliance Control Mapping

1. Import customer-approved catalog versions and preserve external identifiers, text, parameters, provenance, and effective dates.
2. Build scoped profiles with selected controls, enhancements, parameters, environment, tailoring, and justification.
3. Classify crosswalks as `EQUIVALENT`, `PARTIAL`, `SUPPORTING`, `RELATED`, `NOT_EQUIVALENT`, or `UNKNOWN`; similarity alone is never equivalence.
4. Map control to requirement, implementation, test, evidence, and independent assessment.
5. Separate common, system-specific, and hybrid controls and record responsibility inheritance.
6. Require customer-approved sources for parameter values and human approval for tailoring, legal interpretation, and formal audit conclusions.

## Acceptance

Keep catalog and profile versions immutable, audit exclusions, and distinguish control conformance from actual security risk.
