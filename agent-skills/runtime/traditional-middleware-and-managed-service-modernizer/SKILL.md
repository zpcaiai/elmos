# Middleware Modernizer

## Workflow

1. Classify application server, web server, broker, ESB, API gateway, cache, scheduler, file transfer, directory, configuration, and service registry instances.
2. Separate vendor runtime configuration from deployed business applications and business logic.
3. Inventory archives, libraries, domains, clustering, sessions, naming, messaging, transactions, realms, classloaders, startup, admin automation, and licenses.
4. Preserve web virtual hosts, TLS, rewrites, proxying, static assets, compression, authentication, modules, files, and logging semantics.
5. Preserve broker queue/topic durability, ordering, transactions, retry, DLQ, security, persistence, size, and protocol semantics.
6. Extract ESB routes, mapping, orchestration, protocol, business rules, retry, transformation, state, and monitoring rather than translating business flows into mesh configuration.
7. Preserve scheduler calendars, dependencies, credentials, timezone, missed-run behavior, reruns, output, and side effects.
8. Compare upgrade, modern VM, whole-runtime container, application-only container, lightweight runtime, managed service, function extraction, rearchitecture, and retirement.

For managed services, require compatibility, residency, network, performance, availability, responsibility, migration, exit, cost, and license evidence. Include license recovery and exit execution in delivery; do not claim equivalence without tests.
