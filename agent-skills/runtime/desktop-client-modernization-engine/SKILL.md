# Desktop Client Modernization

## Workflow

1. Inventory windows, dialogs, menus, tray, shortcuts, clipboard, print, file/device access, serial/smart-card/COM/native libraries, local databases, installer, signing, deep links, IPC, and update channels.
2. Coordinate .NET runtime/project work with the .NET engine while retaining screen, control, binding, command, visual, storage, and packaging evidence here.
3. For Electron, require context isolation, sandboxing, navigation/window policy, CSP, typed preload APIs, IPC sender validation, allowlisted operations, timeouts, and audit.
4. Compare storage encryption, migration, backup, locks, multi-user behavior, upgrade, downgrade, rollback, and user-data preservation.
5. Prove device capabilities before choosing embedded Web, Web shell, full Web replacement, or native rewrite.

## Gates

Block arbitrary command bridges, remote Node integration, unvalidated IPC, unproven device replacement, unsafe local-data migration, and unsigned/unapproved release operations.

## Output

Emit desktop capability/integration inventories, target decision, bridge contract, storage and packaging results, manual signing/release gates, and Evidence.
