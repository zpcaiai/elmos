# Data Protection and Privacy

1. Record data subject, source, purpose, recipient, region, approved legal or policy basis reference, retention, sharing, model use, and deletion path without making legal conclusions.
2. Test necessity, duplication, retention, service sharing, logs, prompts, analytics, and backup copies for minimization.
3. Choose explicit encrypt, tokenize, pseudonymize, mask, hash, redact, drop, aggregate, or generalize controls.
4. Separate static test, query, BI, log, and display masking; never treat UI masking as API or database authorization.
5. Include source, email, file, object, log, ticket, prompt, clipboard, export, and API in DLP scope.
6. Trace deletion through primary, replicas, caches, search, lakehouse, BI, features, training data, and approved backup/legal-hold policy.

## Acceptance

Bind controls to classification, purpose, lineage, owner, evidence, and customer-approved policy. Route legal judgments to qualified people.
