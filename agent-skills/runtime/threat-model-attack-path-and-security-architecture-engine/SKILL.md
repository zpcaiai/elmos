# Threat Modeling and Security Architecture

1. Bind the threat model to the System Landscape, trust boundaries, identities, assets, data flows, threat intelligence, incidents, and migration plan.
2. Support STRIDE, PASTA, LINDDUN, attack trees, and abuse cases without forcing one method.
3. Record actor, entry, preconditions, technique, impact, likelihood, detection, mitigation, residual risk, and required test for each threat.
4. Convert every trust assumption into a testable, owned, versioned assertion.
5. Visualize attack paths and map mitigations to prevent, detect, respond, and recover controls.
6. Let agents propose candidates and tests only; never delete human threats, mark mitigation complete, accept assumptions, or approve residual risk automatically.

## Acceptance

Trigger reassessment for new endpoints, boundaries, identities, data, providers, models, dependencies, incidents, or major migration changes.
