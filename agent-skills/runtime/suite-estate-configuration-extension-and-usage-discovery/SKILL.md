# Suite Estate Discovery

## Discover

1. Inventory organization, instance, environment, module, process, configuration, extension, data, integration, report, role, batch, and consumer layers.
2. Separate vendor-standard assets from customer configuration, custom code, low-code, reports, integration, and manual shadow processes.
3. Record immutable configuration and metadata exports with product release and environment identity.
4. Correlate runtime trace, audit, jobs, transactions, pages, API, reports, users, and integration observations over a declared window.
5. Classify usage as `ACTIVE`, `SEASONAL`, `DORMANT`, `HISTORICAL`, or `UNKNOWN`; never infer deletion from a short window.
6. Preserve dependency edges and unknown owners or consumers as risk.

## Detect

Flag direct database updates, production-only changes, unversioned configuration, shared admin changes, unknown report consumers, unknown extension owners, and seasonal uncertainty.

## Output

Produce Suite Estate, configuration, customization, extension graph, usage, report, role, and unknown inventories with source and evidence references.
