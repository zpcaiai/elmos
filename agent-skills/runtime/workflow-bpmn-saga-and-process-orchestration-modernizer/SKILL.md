# Workflow, BPMN, and Saga Modernization

## Reconstruct the process

Discover process state across ESB flows, BPM suites, application code, schedulers, databases, checklists, email approvals, and tickets. Classify SHORT_SYNCHRONOUS, LONG_RUNNING, HUMAN_IN_LOOP, SCHEDULED, SAGA, CHOREOGRAPHY, CASE_MANAGEMENT, or UNKNOWN.

Represent BPMN events, tasks, gateways, messages, timers, errors, compensation, subprocesses, and boundary events. Distinguish DOCUMENTATION_ONLY, EXECUTABLE, PARTIALLY_EXECUTABLE, VENDOR_SPECIFIC, and UNKNOWN definitions.

## Preserve recoverability

Record Saga commands/events, state, timeout, retry, compensation, and owner. Persist timers. Preserve human-task role, assignment, SLA, escalation, delegation, audit, and decision. Create an incident with manual recovery when compensation fails.

Build a Process Twin for hidden choreography. Do not confuse a message route with a business process or assume a visual BPMN file is executable.
