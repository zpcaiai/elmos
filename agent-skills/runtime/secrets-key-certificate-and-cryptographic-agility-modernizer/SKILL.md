# Secrets and Cryptographic Agility

1. Scan authorized source, history, build logs, layers, state, configuration, CI, storage, and documentation without copying secret values into evidence.
2. Store only fingerprint, location, type, owner, lifecycle, rotation state, and redacted evidence.
3. Move hard-coded secrets to references, dynamic credentials, then workload identity where supported.
4. On exposure, block use, revoke, rotate, locate downstream exposure, invalidate affected artifacts, notify owners, and preserve audit; deleting text is not revocation.
5. Inventory certificates and algorithms, separate encryption, wrapping, signing, TLS, backup, and root keys, and validate renewal, dual-decrypt, rollback, and destruction windows.

## Acceptance

Reject unowned or unrotatable crypto assets, monitor expiry, apply a versioned algorithm policy, and retain compatibility and rollback evidence.
