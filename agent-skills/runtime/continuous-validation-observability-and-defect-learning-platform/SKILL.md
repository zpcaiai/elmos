# Continuous Validation and Defect Learning

## Define triggers and frequency

Support source, dependency, contract, schema, infrastructure, configuration, model, test, and tool changes plus schedules, incidents, production defects, and expiring exceptions. Schedule on commit, pull request, merge, nightly, weekly, pre-release, post-deploy, continuous synthetic, or incident-triggered frequency based on risk.

## Make tests observable

Record traces, resources, environment, dependencies, test data, duration, retries, failures, artifact identity, and selection reason for every attempt. Measure pass, fail, flaky, duration, queue/setup, retry, mutation, coverage, cost, and defect yield.

## Bound production validation

Allow only approved synthetic monitors, read-only probes, shadow tests, canary assertions, post-deploy smoke, and data invariants. Prevent uncontrolled business side effects and production secret/data exposure.

## Close defect learning

Link a production defect to its failure fingerprint, missing-test analysis, fail-before-fix regression, fix, test promotion, and risk-model update. Record severity, escape stage, root cause, detection gap, and preventive action. Do not close learning while the regression proof is absent.

## Expire evidence automatically

Invalidate results when artifact, test, test data, environment, dependency, contract, framework, tool, or rule changes. Preserve a traceable learning action for every new test, property, contract, fixture, virtual scenario, mutation rule, or gate rule.
