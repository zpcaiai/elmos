# Identity and Zero Trust Modernization

1. Separate human, device, service, workload, privileged, and emergency identities; identify shared accounts, static keys, local users, Basic auth, IP trust, and over-privilege.
2. Prefer federation, phishing-resistant MFA, workload identity, short-lived tokens, resource authorization, and continuous session evaluation.
3. Require recent strong authentication, device posture, risk checks, approval, and JIT grants for privileged operations; audit and auto-revoke sessions.
4. Evaluate subject, device, workload, resource, action, environment, time, risk, authentication strength, session, and data classification for every decision.
5. Time-box identity-aware proxies, token exchange, Kerberos or header bridges; restrict scope and define removal evidence.

## Acceptance

Never grant implicit trust from network location. Require separate least-privilege workload identities and an auditable exit plan for every legacy bridge.
