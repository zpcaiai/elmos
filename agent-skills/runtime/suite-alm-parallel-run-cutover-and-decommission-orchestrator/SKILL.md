# Suite ALM and Cutover

## Promote

1. Version configuration, code, extension, data package, integration, workflow, role, report, infrastructure, and Cutover plan as one Composite Change Set.
2. Promote the same approved artifact through DEV, TEST, SIT, UAT, PREPROD, and PRODUCTION; detect manual production drift.
3. Choose process, company, country, module, user-cohort, read-only, financial, or report parallel mode.

## Cut over

1. Freeze configuration and master data, extract final delta, migrate, reconcile, enable users and integrations, start batch, open period, validate, and enter stability hold.
2. Track legal entity, company, country, plant, business unit, module, user, process, and partner frontiers independently.
3. Model configuration rollback, user routing, integration rollback, data repair, transaction reentry, financial reversal, legacy reactivation, and forward fix.
4. Record point-of-no-return events such as fiscal posting, payroll, tax filing, external invoice, authority switch, period close, or credential revocation.

## Decommission gate

Require zero active legacy users, interfaces, batch writers, and open transactions; migrated reports; accessible audit and history; complete archive; revoked identities and credentials; handled licenses and support; and checked legal holds. Keep Cutover and Decommission as independent decisions.
