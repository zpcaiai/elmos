# Frontend ELMOS Integration

## Workflow

1. Reuse Repository Snapshot, Assessment Run, Migration Plan/Step, Patch Segment, Validation Result, Risk, Approval, Evidence, Delivery Snapshot, Portfolio, Cost, and Audit authorities.
2. Wrap frontend artifacts in `elmos.frontend-client-evidence.v1` with tenant, Snapshot, engine/policy/schema versions, artifact reference, content hash, creation time, and upstream Evidence references.
3. Map route removal, visual regression, accessibility violation, auth regression, storage incompatibility, and unsupported old client versions to stable cross-engine Risk categories.
4. Build a Composite Change Set linking frontend, BFF, backend, design-system, desktop, Android/iOS, and release-configuration changes without duplicating ownership.
5. Meter scan, codemod, browser, visual, accessibility, desktop, Android, iOS, and Agent repair usage through existing commercial authorities.

## Gates

Reject empty Evidence, cross-tenant references, mutable artifacts, missing hashes, unsupported schema versions, self-approved visual/accessibility outcomes, and release/decommission claims without provider observations.

## Output

Emit unified Evidence extensions, Risk mappings, Portfolio topology, Change Set, cost events, audit events, Delivery references, and offline-verifiable Evidence Pack entries.
