# Semantic Obligation Registry
Read `../references/uir-v1.md`. Create stable obligations for types, absence, numerics, order, exceptions, async, cleanup, transactions, concurrency, serialization, APIs, effects, security, performance and dynamic/manual work. Link source/UIR scope, statement, source semantics, target requirement, severity, status, evidence and at least one verification strategy. Blocking obligations cannot disappear into logs; waiver requires accountable reason and audit.
