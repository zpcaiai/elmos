# Async and Concurrency Semantics Modeler
Read `../references/uir-v1.md`. Preserve source async kind, hot/cold/start behavior, error channel, backpressure, suspend/resume, callback registration versus dispatch, scheduler/context affinity, cancellation/timeout/cleanup, blocking, memory ordering, locks and atomics. Never equate Promise/Task/Future, generator/collection, or async/sync. Unknown concurrency cannot imply thread safety.
