# Skill 1308: b37-framework-adapter-sdk

## Use this skill when

- A new source or target framework must integrate without hard-coding it into the core engine.
- Partners need certified framework packs.

## Domain-specific risks and invariants

- Annotation-only mappings miss lifecycle security transaction and runtime behavior.
- Provider-specific behavior can be mistaken for generic framework semantics.

## Workflow

1. Define static and runtime fingerprint contracts.
2. Expose Framework Contract Model builders and target-profile interfaces.
3. Provide typed provider mapping and recipe APIs with obligations.
4. Create reference adapter and conformance corpus covering web DI config validation security persistence transactions messaging cache scheduler and lifecycle.
5. Run real startup and P0 contract tests for claimed capabilities.

## Required repository outputs

- framework-adapter SDK and sample pack
- `FCM and target-profile schemas/types`
- runtime fingerprint and contract conformance evidence

## Verification

- Validate source traces for every FCM fact.
- Start real source and target applications.
- Reject provider or lifecycle capabilities without evidence.

## Stop and escalate when

- Security transaction or persistence semantics are unknown.
- The adapter reads framework internals without a supported compatibility contract.

## Definition of done

- One framework adapter is externally implementable and certified at a bounded scope.
- No silent framework drops exist.
- Provider-specific limitations are explicit.
