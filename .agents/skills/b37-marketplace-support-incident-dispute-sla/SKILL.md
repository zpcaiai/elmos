# Skill B37-X10: b37-marketplace-support-incident-dispute-sla

## Use this skill when

- The Batch 37 extension ecosystem requires this lifecycle capability to be production-complete and independently testable.
- A Codex implementation task must create typed contracts, deterministic workflows, evidence, and conservative gates rather than prose-only policy.

## Domain-specific risks and invariants

- Support access follows least privilege, time limits, customer approval, and audit.
- Publisher contracts cannot disclaim marketplace security or revocation responsibility.
- Refund or commercial disputes never bypass technical containment.

## Workflow

1. Define L1/L2/L3 ownership among customer, marketplace operator, publisher, and platform engineering.
2. Define severity, response, update, mitigation, resolution, escalation, and communication SLAs.
3. Generate privacy-filtered diagnostic bundles bound to tenant, extension digest, environment, and incident.
4. Implement secure JIT support access, publisher escalation, marketplace takeover, refund/dispute routing, and publisher non-response handling.
5. Run support simulations for outage, data corruption, security incident, billing dispute, and abandoned extension.

## Required repository outputs

- `support/support-policy.json`
- diagnostic bundle schema, SLA records, escalation matrix, dispute cases, and takeover evidence

## Verification

- Run `validate_legal_support.py`.
- Demonstrate end-to-end incident routing, secure diagnostics, SLA timers, escalation, and closure.

## Stop and escalate when

- No accountable responder exists for P0 incidents.
- Resolution requires unrestricted access to customer source or secrets.

## Definition of done

- Support, incident, dispute, refund, and abandoned-publisher cases have tested accountable workflows.
